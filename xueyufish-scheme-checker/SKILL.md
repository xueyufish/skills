## Metadata
name: scheme-checker
description: 这是一个用于检查和评审方案文档是否符合我的要求的资源。每当被要求检查方案材料是否符合我的预期时，需要使用此 Skill。

## When to use this skill
当被要求检查和评审方案是否符合我的要求时使用这一技能，包括：
- 需求设计方案
- 架构设计方案
- 开发设计方案
- FAQ 答疑
- 项目状态报告
- 风险报告

## How to use this skill
检查文档材料时：
1. **从请求中识别文档类型**
2. **从 `examples/` 目录中加载相应的模板文件**：
   - `examples/arch-designs.md` - 用于开发方案/架构设计方案/需求设计方案/数据设计方案等研发设计方案
   - `examples/faq-answers.md` - 用于回答常见问题
   - `examples/general-comms.md` - 用于其他未明确匹配上述任何一种类型的沟通内容
3. **遵循该文件中的具体说明** 以确定格式、语气和内容收集方式。

如果文档类型与现有的任何指南都不匹配，请求澄清或获取更多关于所需格式的上下文信息。
